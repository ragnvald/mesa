Welcome to the MESA 4.0 tool. The tool is based on a method developed by UNEP-WCMC and the Norwegian Environment Agency. You are now in the quick guide for our tool. Our aim with this page is to give you some basic information about how to use the system.

Import data

To work with MESA you will have to provide input data. This is part of building up the database. Eventually your input data will be processed and you will be provided with a database for visualisation or further analysis.

Valid data formats are shapefiles or a geopackage file with tables. The data must be placed in the correct folders:

Data Folder Comment
Asset input/asset Spatial format can be point, lines or polygons
Geocode input/geocode Calid format is polygons
Atlas input/atlas Valid format is polygons
Lines input/lines Lines are only valid data format

Set up processing

Processing data for a sensitivity atlas involves registering asset values and suceptibilitis. This is done by pressing the Set up-button.

Process data data

This is where you initiate the processing. Processing time will tepend on how many assets you have and the granularity of your geocode objects. It could take anything a couple of minutes to an hour.

Work with lines

If you would like to work with line segments you will need to store appropriate lines in the lines-folder. You will have three options in this user interface.

  • Initiate - creates some random lines for you to work on
  • Edit segments - here you can edit the width and length of the line segments per line. This means you can have individual lengts and withs.
  • Process segments does the same for line segments as the processing does for the geocode objects. Map and analysis layers are created.
  • Lines (input/lines)

The system can also generate some random lines for you. You can then adjust these lines later.

Making maps

We have prepared a QGIS project file. The file contains prepared styles and map templates making it dtraight forward to produce a sensitivity atlas.

Since all MESA-data is stored in a database file other desktop or server tools may be used. It will however require detailed knowledge about the GIS-tool.

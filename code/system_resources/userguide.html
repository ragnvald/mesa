Welcome to the MESA 4.0 tool. The tool is based on a method developed by UNEP-WCMC and the Norwegian Environment Agency. You are now in the quick guide for our tool. Our aim with this page is to give you some basic information about how to use the system.

User interface overview

The MESA user interface can be divided into two types; imports/processing and data entry/edit. When data is imported or processed you will find that the log window provides information about the progress.

Aprogress bare will give you a visual indication of the progress. It however may not always represent an accurate account of the progress.

When you edit data, move to the next record or close the dialogue, your edits will be automatically saved.

Import data

To work with MESA you will have to provide input data. This is part of building up the database. Eventually your input data will be processed and you will be provided with a database for visualisation or further analysis.

Valid data formats are shapefiles or a geopackage file with tables. The data must be placed in the correct folders:

Data Folder Comment
Asset input/asset Spatial format can be point, lines or polygons
Geocode input/geocode Calid format is polygons
Atlas input/atlas Valid format is polygons
Lines input/lines Lines are only valid data format

If you import new data after having processed your data you will have to enter the setup data again.

Set up processing

Processing data for a sensitivity atlas involves registering asset values and suceptibilitis. This is done by pressing the Set up-button.

Process data data

This is where you initiate the processing. Processing time will depend on how many assets you have and the granularity of your geocode objects. It could take anything a couple of minutes to an hour.

Work with lines

If you would like to work with line segments you will need to store appropriate lines in the lines-folder. You will have three options in this user interface.

  • Initiate - creates some random lines for you to work on
  • Edit segments - here you can edit the width and length of the line segments per line. This means you can have individual lengts and withs.
  • Process segments does the same for line segments as the processing does for the geocode objects. Map and analysis layers are created.
  • Lines (input/lines)

The system can also generate some random lines for you. You can then adjust these lines later.

Making maps

We have prepared a QGIS project file. The file contains prepared styles and map templates making it straight forward to produce a sensitivity atlas.

Since all MESA-data is stored in a database file other desktop or server tools may be used. It will however require deailed knowledge about the GIS-tool.

Resetting the system

At any time, if you want to give your project a fresh start, you can do so by deleting the existing database. This is the procedure:

  1. Stop this program by pressing the Exit-button
  2. Delete the database file. Its name is mesa.gpkg. You will find it in the folder /output

Handling your data

National regulations might stop you from sending your data abroad for processing. In MESA 3.0 we facilitated for cloud based processing. In this version we are back on processing on the desktop this way you will always have your data with you.

The system will by default send usage statistics to an online server. On the Registration-page you may turn of this functionality. Statistics are sent when you start up the system. It will only do this once every 24th hour. If you do not have any internet access the statistics will not be sent.

An open system for all

MESA 4.0 is open source code. It uses open source libraries and will not require any paid licenses for use.

If you would like to look at the code or even contribute with comments, bug messages or even updates to the code you can do this on GitHub.
